# AGENTS.md

Guidance for AI agents (Claude Code, Codex, etc.) working in this repository.

## Repository Overview

This is a personal **learning-notes vault** — plain-text Markdown notes in
[Logseq](https://logseq.com/) outliner format. It is **not a software project**:
there is no code, build, lint, or test step. Each file is a self-contained note
summarizing something the owner is learning (a paper, an article, a technical
concept, hands-on exploration).

The repo is cloned as a child directory of the `Logseq-github` vault but is its
own independent Git repo (`git@github.com:hczhu/learning-notes.git`), gitignored
by the parent.

## File Naming Convention

```
YYYY-MM-kebab-case-topic.md
```

- Prefix with the year-month of the source material or when the note was created,
  whichever is more meaningful (e.g. `2026-07-listening-the-profession-of-it.md`,
  `2023-02-toolformer-lms-can-teach-themselves-to-use-tools.md`).
- Day is normally omitted (month granularity is enough).
- One topic per file; keep the slug short and descriptive.

## Logseq Markdown Format

Notes use the Logseq outliner style — everything is a bullet, nesting uses tabs:

```markdown
- **Source**: <attribution / link> (for notes derived from a paper or article)
- **One-liner**: <the single key takeaway>

- ## Section Title
	- Content line
		- Nested detail
	- **Bold label**: value
```

Conventions seen across existing notes:
- A top `## Section Title` (or a `- **Source**:` / `- **One-liner**:` header for
  source-derived notes) — no `tags::` front matter is required in this repo.
- **LaTeX math** via `$inline$` and `$$display$$` blocks.
- **Markdown tables** for dense comparisons (e.g. the GPU-connectivity note).
- **Image embeds**: `![alt](../assets/<file>)` or external URLs. Note that
  `../assets/` points at the Logseq vault's shared assets folder, which lives
  **outside** this repo — those image paths may not resolve from a bare clone.

## GitHub + Logseq Dual Rendering

Notes must render correctly **both** in Logseq (outliner) and on GitHub
(CommonMark/GFM). GitHub treats tab-indented lines that are not nested under a
list item as **indented code blocks** (monospace boxes) — the most common way an
outline file breaks on GitHub.

1. **Headings must be bullets**: write `- ## Section Title`, never a bare
   `## Section Title` followed by tab-indented children — the children render as
   a code block on GitHub.
2. **Only tab-indent under a parent bullet** — one tab per nesting level; no
   free-standing tab-indented text.
3. **Tables hang off a bullet**: start the table on the bullet line
   (`- | Col | Col |`) and indent subsequent rows with the same tabs plus two
   spaces so the `|` characters align vertically. A table on bare tab-indented
   lines (no bullet) becomes a code block on GitHub.
4. **Currency dollar signs**: escape as `\$` when two or more `$<digit>`
   patterns appear on one line — paired `$...$` renders as inline math and
   garbles the text. (`$inline$` used *intentionally* for LaTeX is fine.)
5. After merging, spot-check the rendered note on GitHub when it contains new
   tables or deep nesting.

## Writing a New Note (agent instructions)

- When asked to "create a learning note" from a source (PDF, article, transcript),
  **extract the key insights, facts, and data points** — do not transcribe
  verbatim. Prefer structured bullets and tables over prose.
- Always include a `- **Source**:` line when the note is derived from external
  material, with enough detail to find the original (author, title, venue, date,
  DOI/URL).
- **No images in notes generated by agents** — if the source is a PDF/screenshot,
  OCR/extract the text and work from that; do not embed the image.
- Match the existing Logseq outliner style and the file-naming convention above.

## Git Workflow

- **Never push directly to `main`.** Always create a branch, open a PR, and merge
  it (this matches the workspace-wide rule in the parent `tony-stock` repo).
- Non-code changes (new notes, edits) may be committed and PR'd without prior
  confirmation.
- Typical flow:
  ```bash
  git checkout -b <branch>
  git add <note>.md
  git commit -m "<message>"
  git push -u origin <branch>
  gh pr create ... && gh pr merge --squash --delete-branch
  ```
